20140679 전산학부 김재성

테스트 환경 : macOS sierra
개발 언어 : Python 3.5.2
컴파일 방법 : 필요 없습니다.
실행 방법 : python main.py [e-nfa 파일경로]

입출력 예시
main.py와 같은 경로에 e-nfa2.txt 가 존재하는 경우(내용은 프로젝트 안내에 나온 파일과 동일)
    > python main.py e-nfa2.txt
로 실행시키면 폴더 내에 output_e-nfa2.txt가 생성되고 그 내용은 e-nfa2.txt의 e-nfa 를 m-dfa 로 변환한 결과물입니다.

스크린샷은 사진으로 별도 첨부.
실행화면1.png : 터미널에서의 main.py 실행 방법입니다. 표준 출력이 아니라 파일 출력으로 결과를 생성하기 때문에 실행결과 화면은 디버깅용 프린트문 뿐입니다.

그 외 파일들
e-nfa2.txt : 임의로 만든 예시 인풋입니다.
output_e-nfa2.txt : 위 파일에 대하여 main.py 를 실행시킨 결과입니다.

구현 정책(중요합니다)
1. 모든 스테이트는 q0, q1, ...의 형태(q+0부터 시작하여 연속된 정수)로 쓰여진다고 가정했습니다. 따라서 A, B, C, .../ a,b,c,.../ 1,2,3,.../ d0, d1, d2, ... 등 형식에 맞지 않는 스테이트가 인풋으로 들어올 경우 에러가 납니다.
2. 결과 파일도 스테이트를 q0, q1, ... 로 나타내었습니다. q뒤에 붙는 숫자는 0부터 시작하여 연속된 정수입니다.
3. dead state는 제외하도록 구현했습니다. 따라서 dead state를 포함시켜 구현한 것보다 결과의 스테이트 개수가 하나 작습니다.
